# Onicode vs OpenCode / Cascade — Gap Analysis

> Generated 2025-03-09 after studying [OpenCode](https://github.com/anomalyco/opencode) docs, agents, tools, and comparing with Cascade/Windsurf capabilities.

---

## 1. Agent Architecture

| Feature                     | OpenCode                                                  | Cascade/Windsurf           | Onicode                                             | Gap                                          |
| --------------------------- | --------------------------------------------------------- | -------------------------- | --------------------------------------------------- | -------------------------------------------- |
| Primary agents (build/plan) | ✅ `build` (full access) + `plan` (read-only)             | ✅ Single agent with modes | ❌ Single agent, no modes                           | **HIGH** — Need at least build/plan split    |
| Subagents                   | ✅ `general` (parallel work) + `explore` (fast read-only) | ✅ Fast Context subagent   | ❌ `spawn_sub_agent` stub exists but never executes | **CRITICAL** — Sub-agents don't actually run |
| Context compaction          | ✅ Auto-compacts long conversations                       | ✅ Automatic               | ❌ None — hits context limit and fails              | **CRITICAL** — Long sessions will break      |
| Auto-continuation           | ✅ Built into agent loop                                  | ✅ Built-in                | ✅ Added in v0.7.3                                  | Fixed                                        |
| Max tool rounds             | ✅ Configurable                                           | ✅ ~100+                   | ✅ 50 rounds                                        | OK                                           |
| Session titles              | ✅ Auto-generated by title agent                          | ✅ Auto-generated          | ❌ First message used as title                      | **LOW** — Nice to have                       |
| Session summaries           | ✅ Auto-generated by summary agent                        | ✅ Automatic               | ❌ None                                             | **MEDIUM**                                   |

## 3. Project Context & Understanding

| Feature                      | OpenCode                                                | Cascade               | Onicode                                        | Gap                                                    |

## Priority Roadmap — Status

### P0 — Must Fix Now

1. ~~**Tasks tab sync**~~ ✅ DONE — TasksTab fetches from TaskManager IPC, maps to Kanban columns
2. ~~**Sub-agent execution**~~ ✅ DONE — Auto-continuation loop serves the same purpose (up to 5 auto-continues)
3. ~~**Context compaction**~~ ✅ DONE — `compactConversation()` auto-summarizes when conversation exceeds ~80% of token budget

### P1 — Required for Parity

4. ~~**webfetch tool**~~ ✅ DONE — HTTP fetch with HTML stripping, redirect following, 15s timeout
5. ~~**websearch tool**~~ ✅ DONE — DuckDuckGo HTML lite parsing, no API key needed
6. ~~**Fast context / explore agent**~~ ✅ DONE — `explore_codebase` tool analyzes structure, deps, entrypoints, config, tech stack
7. ~~**AGENTS.md auto-generation**~~ ✅ DONE — `init_project` now creates AGENTS.md with project overview
8. **LSP integration** — ⏳ DEFERRED — Requires language server setup; agent can use `search_files` + `read_file` for now

### P2 — Important

9. ~~**Glob tool**~~ ✅ DONE — `glob_files` with git ls-files (respects .gitignore) or find fallback
10. **Skills system** — ⏳ DEFERRED — Can be added later as SKILL.md loader; agent can use `memory_write` for now
11. ~~**Permissions system**~~ ✅ DONE — `DEFAULT_PERMISSIONS`, `loadProjectPermissions()`, `checkPermission()`, per-project `.onicode/config.json`
12. ~~**Plan agent mode**~~ ✅ DONE — `setAgentMode('plan')` denies all writes, asks for commands. IPC: `agent-set-mode`, `agent-get-mode`
13. ~~**.gitignore respect**~~ ✅ DONE — `search_files` prefers ripgrep (respects .gitignore). `list_directory` expanded skip list. `glob_files` uses git ls-files.

### P3 — Nice to Have

14. ~~**Session titles**~~ ✅ DONE — `generateSessionTitle()` fires on first user message, sends `ai-session-title` event
15. **Session summaries** — ⏳ DEFERRED — Can be added as compaction agent extension
16. **Deploy tool** — ⏳ DEFERRED — Requires Netlify/Vercel API integration
17. ~~**Config file**~~ ✅ DONE — `loadProjectPermissions()` reads `.onicode/config.json` for per-project settings
18. **Conversation sharing** — ⏳ DEFERRED — Requires backend infrastructure
